# Recent X Activity Output Schema

This document defines the project-owned Recent X Activity evidence contract for `Enrich_XAUUSD_Leads_Full`.

The helper is intentionally narrow:

- X source only.
- JSON output only.
- No markdown.
- No HTML.
- No LLM calls.
- No CSV writes.
- No broad multi-source research.

## Schema

```json
{
  "schema_version": "recent-x-activity/v1",
  "username": "@goldtrader",
  "query": "@goldtrader XAUUSD gold trading",
  "window_days": 30,
  "status": "completed",
  "source": "xquik",
  "fetched_at": "2026-06-27T10:00:00Z",
  "cache_hit": false,
  "posts": [
    {
      "text": "XAUUSD swept liquidity before NY open...",
      "created_at": "2026-06-25T14:10:00Z",
      "url": "https://x.com/example/status/123",
      "like_count": 42,
      "reply_count": 5,
      "repost_count": 8
    }
  ],
  "evidence": {
    "recent_activity": true,
    "xauusd_mentions": 1,
    "gold_mentions": 0,
    "trading_terms": ["xauusd", "liquidity"],
    "themes": ["ICT", "Gold only"],
    "persona_hint": "ICT Trader",
    "summary": "Recent X activity suggests ict trader with themes: ICT, Gold only; across 1 XAUUSD and 0 gold mention(s)."
  },
  "warnings": []
}
```

## Field Definitions

| Field | Type | Description |
|---|---|---|
| `schema_version` | string | Always `recent-x-activity/v1`. |
| `username` | string | Normalized display username with leading `@`. |
| `query` | string | Human-readable query passed by Hermes for auditability. |
| `window_days` | number | Recent activity window, normally `30`. |
| `status` | string | `completed`, `failed`, or `skipped`. |
| `source` | string | `xquik` when Xquik was used, otherwise `none`. |
| `fetched_at` | string | UTC timestamp when this payload was produced. |
| `cache_hit` | boolean | `true` when same-day cache was reused. |
| `posts` | array | Compact normalized X posts authored by the requested username only. |
| `evidence` | object | Deterministic keyword-based evidence summary computed only from target-authored posts. |
| `warnings` | array | Non-fatal diagnostics. |

Hermes must validate `schema_version == "recent-x-activity/v1"` before using helper output. If the value is missing or different, treat the lookup as failed and continue with CSV-only evidence.

## Status Values

| Status | Meaning |
|---|---|
| `completed` | Xquik was called successfully or returned a usable response. May still have empty `posts`. |
| `skipped` | Lookup did not run, usually because `XQUIK_API_KEY` is not set. |
| `failed` | Timeout, request error, JSON parse error, invalid username, or unexpected failure. |

## Evidence Rules

Evidence is deterministic. It is not generated by an LLM.

Evidence must use only tweets verified as authored by `username`. Returned Xquik records from other authors, mentions of the account, quoted tweets by other users, and nested non-target posts are discarded before evidence extraction. If all returned posts are discarded, `status` remains `completed`, `posts` is empty, `recent_activity` is `false`, and `summary` is empty.

`status: "completed"` with `posts: []` is valid and must not be treated as a failure. It means no target-authored recent posts were found. Hermes should not fabricate evidence; it should keep CSV-only enrichment and preserve the completed audit status.

Helper `warnings` and `posts` are JSON-only audit data. They must not be copied wholesale into `enriched_leads.csv`. The final CSV contract remains exactly:

```text
Name
Username X
First-line cá nhân hóa
Score fit
Hook
```

The helper counts:

- `xauusd_mentions`
- `gold_mentions`
- trading keyword matches
- theme keyword matches
- one `persona_hint`

Theme labels:

- `ICT`
- `SMC`
- `Price Action`
- `Scalping`
- `Swing`
- `Prop Firm`
- `Signal Provider`
- `Gold only`
- `Multi-asset`

Persona hints:

- `ICT Trader`
- `SMC Trader`
- `Price Action Trader`
- `Gold Analyst`
- `Swing Trader`
- `Scalper`
- `Prop Trader`
- `Educator`
- `Signal Provider`
- `Macro Trader`
- `Hybrid`

## Failure Contract

Failures must still return schema-compatible JSON:

```json
{
  "schema_version": "recent-x-activity/v1",
  "username": "@goldtrader",
  "query": "@goldtrader XAUUSD gold trading",
  "window_days": 30,
  "status": "failed",
  "source": "xquik",
  "fetched_at": "2026-06-27T10:00:00Z",
  "cache_hit": false,
  "posts": [],
  "evidence": {
    "recent_activity": false,
    "xauusd_mentions": 0,
    "gold_mentions": 0,
    "trading_terms": [],
    "themes": [],
    "persona_hint": "",
    "summary": ""
  },
  "warnings": ["xquik request failed: ..."]
}
```

Hermes must treat failed/skipped research as non-blocking and continue with CSV-only evidence.
